# 資訊及通訊科技（ICT）教科書B冊 筆記總覽

## 單元 B：電腦系統基礎

### 第 1 章：輸入及輸出設備

本章節的學習目標包括描述輸入、處理與輸出之間的關係，識別及比較常見輸入與輸出設備的功能和規格，並展示其在日常生活中的應用。

### 1.1 輸入-處理-輸出周期 (IPO Cycle)

電腦處理數據的基本過程稱為**輸入-處理-輸出周期/循環 (Input-Process-Output Cycle, IPO Cycle)**。

| 步驟 | 描述 | 來源依據 |
| :--- | :--- | :--- |
| 1. **輸入 (Input)** | 將**數據**送入電腦。 | |
| 2. **處理 (Processing)** | 程式處理所輸入的數據，儲存的數據亦可在這一步驟被提取或更新。 | |
| 3. **輸出 (Output)** | 電腦輸出有意義的**資訊**。 | |
| **儲存 (Storage)** | 儲存設備（如硬碟）用於**儲存**處理後的資訊或供處理用的數據。 | |

### 1.2 輸入設備 (Input Devices)

輸入設備負責 IPO 循環的第一步，用於向電腦輸入指令，或從現實世界中擷取數據並傳輸到電腦。

#### A. 連接方式

輸入設備可使用以下方式連接到電腦：

*   **有線連接**：通用串列匯流排 (USB) 連接埠。
*   **無線連接**：射頻 (RF) 技術（如 2.4 GHz 連接）、**藍牙 (Bluetooth)** 或 **Wi-Fi 連線**。

#### B. 常用輸入設備類型

| 類型 | 設備名稱 | 描述 / 備註 | 來源依據 |
| :--- | :--- | :--- | :--- |
| **文字輸入** | **鍵盤 (Keyboard)** | 標準鍵盤通常有 100 個或更多按鍵。常見的鍵盤配置是 **QWERTY**。數字鍵盤常用於銀行或銷售點終端機。 | |
| **動作追蹤** | **電腦滑鼠 (Mouse)** | 追蹤其在平面上的移動以控制游標。現時多數為**光學滑鼠**（LED 或激光）。 | |
| | **追蹤球 (Trackball)** | 一個安裝在凹槽中的滾球，滾動它來控制游標。適用於需要較少空間或位置固定的情況。 | |
| | **操控桿/賽車軚盤** | 用於遊戲和培訓駕駛者。 | |
| **位置追蹤** | **觸控板 (Touchpad)** | 常見於手提電腦，測量用戶手指的動作和壓力。有些支援**多點觸控 (multi-touch)**。 | |
| | **觸控式屏幕 (Touchscreen)** | 既是輸入也是輸出設備，是觸控板與顯示器的結合。相比滑鼠，其優點包括手眼協調更佳、更易操作、所需空間少，但**生產成本較高**。 | |
| **聲音輸入** | **麥克風 (Microphone)** | 用於創設數碼音頻檔案。 | |
| | **MIDI 鍵盤** | 將音符或數碼化的聲音樣本輸入至 MIDI 檔案中，不受環境噪音影響。 | |
| **圖像/視像** | **圖像掃描器 (Scanner)** | 用於建立數碼圖像檔案，平面圖像掃描器最為常見。 | |
| | **光符識別 (OCR)** | 將掃描的圖像轉換為文字檔案，以便進一步處理和數據分析。 | |
| | **數碼相機/錄影機** | 利用影像感應器捕捉真實世界的影像或影片。 | |
| | **網絡攝影機 (Webcam)** |  usually directly connected to the computer, primarily used for video conferencing and video surveillance. | |
| **自動輸入** | **光學閱讀器** | 掃描可視數據並將其轉為數碼數據。 | |
| | **條碼掃描器 (Barcode Reader)** | 掃描條碼並將其轉換為數字序列或字符串傳送至電腦。條碼常見於產品、優惠券、書籍等。 | |
| | **二維碼掃描器 (QR Code Reader)** | 讀取二維碼，二維碼代表的字符不僅可以是數字序列，還可以包含任何類型的字符。 | |
| **感應器** | **感應器 (Sensor)** | 能探測光、熱、濕度等訊號，並將資訊傳遞給其他設備。感應器也被稱為**換能器 (Transducer)**。 | |
| | **常見感應器** | 包括光感應器（亮度）、超聲波感應器（距離）、陀螺儀（定向）、溫度/濕度計。 | |
| **生物識別** | **生物識別設備** | 用於掃描及讀取人體的身體或行為特徵（如指紋、臉部、虹膜），用於身份驗證。 | |

### 1.3 輸出設備 (Output Devices)

輸出設備負責 IPO 循環的最後一步，可以透過輸出設備輸出相片、實體檔案、劇本或聲音等資訊。

#### A. 顯示器 (Monitors)

顯示器是電腦相當重要的輸出設備，用於顯示使用者介面及處理結果。

**常見顯示器類型比較**：

| 類型 | 運作方式 | 優點 |
| :--- | :--- | :--- |
| **液晶體顯示器 (LCD)** | 透過冷陰極螢光燈 (CCFL) 或 發光二極管 (LED) 作為背光。 | 價格較低；屏幕尺寸選擇多。 |
| **有機發光二極管 (OLED)** | **OLED 為光源**，亦能製造顏色，**不需要背光**。 | 對比度更高；亮度更一致；視角更廣；更輕薄。 |

**顯示器規格**：

*   **屏幕尺寸**：通常以對角線的長度量度（以吋為單位）。
*   **解像度 (Resolution)**：即屏幕上有多少點/像素（Pixel）。
*   **像素密度 (Pixel Density)**：指指定範圍中有多少像素，以 **ppi (每吋像素)** 為單位。
*   **長寬比 (Aspect Ratio)**：屏幕寬度與高度的比例（如 16:9, 4:3）。
*   **更新率 (Refresh Rate)**：屏幕在特定時間內可以更新影像的次數，以**赫茲 (Hz)** 為單位。
*   **常見連接埠**：VGA、DVI、**HDMI**（支援音訊，解像度高）和 **DisplayPort**（支援音訊，解像度最高）。

#### B. 打印機 (Printers)

打印機是列印實體副本的設備。

**三種常見打印機比較**：

| 屬性 | 熱感式打印機 (Thermal) | 噴墨打印機 (Inkjet) | 激光打印機 (Laser) |
| :--- | :--- | :--- | :--- |
| **購買價格** | 較便宜 | 較貴 | **非常昂貴** |
| **每頁打印成本** | 較便宜 | 較昂貴 | **較便宜** |
| **打印速度** | 較快 | 較慢 | **較快** |
| **耐用性** | **不適合長期文件**（文字和圖像會隨時間褪色）。 | 如果弄濕，圖文可能會變模糊。 | **不易受水影響**，成品更耐用。 |
| **其他應用** | **常見於收據**，體積小、輕巧、安靜。 | 廣泛用於家居及商業用途。 | 辦公室常用，每頁成本較低。 |

**其他打印機**：包括繪圖儀 (Plotter)（用於繪製高質素向量圖）、**3D 打印機**（按數碼 3D 模型打印出實體）。

#### C. 音頻輸出設備

*   **揚聲器 (Speaker)**：優點為讓大眾聽到聲音，營造環境。
*   **頭戴式耳機/入耳式耳機 (Headphones/Earphones)**：優點為私密度高，對他人的干擾少，更輕便於攜帶，受環境噪音影響較少。

### 1.4 電腦應用

*   **銷售點終端機 (POS Terminals)**：通常由多個輸入設備組成，例如磁帶卡閱讀器/智能卡閱讀器、數字鍵盤、條碼掃描器、二維碼掃描器，以及收據打印機（通常為熱感式）。
*   **商業用途**：視像會議、收銀服務、電子支付系統、自助終端機。
*   **教育用途**：網上課程、圖書館借還、博物館展覽。

### 第 2 章：電腦硬件 (Computer Hardware)

本章節描述電腦硬體，包括主機、處理器、儲存設備和電腦種類。

### 2.1 主機 (Main System)

主機內有電腦系統的主要部件，包括主機板、處理器、儲存設備、電源供應器等。

#### A. 主機板 (Motherboard)

主機板是設有插槽和連接埠的電路板，可供連接各種電子部件，如中央處理器 (CPU)、隨機存取記憶體 (RAM)、圖形卡和輔助儲存設備。

#### B. 處理器 (Processor)

1.  **中央處理器 (CPU)**：電腦的「**大腦**」，負責處理數據和指令，控制電腦系統運作。
    *   CPU 每次處理的位元數目稱為**字長**（一般為 32 位元或 **64 位元**）。

2.  **圖形處理器 (GPU)**：專門負責**圖像處理**，具有更多核心，因此多工作業能力較強。
    *   **獨立圖形卡 (Graphics Card)**：處理能力較高，適用於電腦輔助設計、剪輯影片或電腦遊戲。
    *   **集成圖形處理器 (iGPU)**：集成於主機板或 CPU 內，成本較低，消耗較少能源，適用於一般文書處理。

#### C. 主機內的數據傳輸（匯流排/匯流排系統 Bus System）

匯流排用以連接電腦各部件，分為三種：

| 匯流排類型 | 傳輸的物件 | 方向 | 來源依據 |
| :--- | :--- | :--- | :--- |
| **控制匯流排** | 控制訊號 | **單向**（處理器發送，不接收）。 | |
| **位址匯流排** | 記憶體位址 | **單向**。 | |
| **數據匯流排** | 數據或指令 | **雙向**（發送和接收）。 | |
| **通用串列匯流排 (USB)** | 用作連接**外部設備**（如鍵盤、滑鼠、打印機、外部儲存）。 | |

### 2.2 中央處理器 (CPU)

#### A. 機器周期 (Machine Cycle)

CPU 執行指令的過程稱為機器周期（或稱指令周期），通常包含四個階段：

1.  **提取 (Fetch)**：從 RAM 或快取中獲取指令。
2.  **譯碼 (Decode)**：將指令譯碼，由 **控制部件 (CU)** 負責。
3.  **執行 (Execute)**：執行指令，由 **算術及邏輯運算部件 (ALU)** 負責。
4.  **儲存 (Store)**：將處理結果儲存在**累加器**等寄存器中。

#### B. CPU 性能指標

1.  **時鐘頻率 (Clock Rate)**：CPU 在一秒內進行的時鐘周期數目，單位為赫茲 (Hz) 或 千兆赫 (GHz)。
2.  **核心數 (Core)**：指 CPU 內的處理器單元數目，核心數越多，可同時執行的程序數目越多，性能通常更佳。
3.  **字長 (Word Length)**：CPU 每次處理的位元數，現時常見為 64 位元。

### 2.3 儲存設備 (Storage Devices)

#### A. 主記億體 (Main Memory)

| 類型 | 易失性 (Volatility) | 可重複性 (Re-programmability) | 用途 | 來源依據 |
| :--- | :--- | :--- | :--- | :--- |
| **隨機存取記憶體 (RAM)** | **易失性**（電腦關機時數據消失）。 | 可以重複讀寫。 | 暫存數據供 CPU 處理。 | |
| **唯讀記憶體 (ROM)** | **非易失性**（電腦關機時數據仍會保存）。 | 通常不可重複寫。 | 儲存**啟動電腦所需的一系列指令** (即 BIOS)。 | |

*   **快取記憶體 (Cache Memory)**：位於 CPU 中，儲存速度比 RAM 更快，用於暫存最常存取的數據或指令，以提高效率。

#### B. 輔助儲存器 (Secondary Storage)

所有輔助儲存器都是**非易失性**的。

| 儲存媒介 | 介質 | 存取方式 (Access Method) | 數據存取速度 | 耐用度 | 來源依據 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **磁帶 (Magnetic Tape)** | 磁 | **順序存取**（讀取特定數據前必須先處理所有項目）。 | **最低**。 | 若放置於潮濕環境，則容易受損。 | |
| **硬碟 (HDD)** | 磁 | **直接存取**（亦稱隨機存取）。 | 中。 | 易受撞擊損壞。 | |
| **固態硬碟 (SSD)** | 快閃記憶體 | 直接存取。 | **高**。 | 耐用。 | |
| **光碟 (Optical Disc)** | 光 | 直接存取。 | 中。 | 刮花時容易受損。 | |

*   **記憶體階層 (Memory Hierarchy)**：儲存設備按數據存取速度、儲存容量和每容量價格分層。
    *   位置越高（例如寄存器、快取），存取速度越高，容量越小，每容量價格越高。
    *   順序：**寄存器** → **處理器的快取記憶體** → **RAM** → **SSD** → **硬碟** → **磁帶**。

### 第 3 章：電腦軟體 (Computer Software)

電腦軟體是數據或指令，用作協助電腦完成特定工作。

### 3.1 電腦系統

軟體分為兩類：**系統軟體** 和 **應用軟體**。

| 屬性 | 系統軟體 (System Software) | 應用軟體 (Application Software) | 來源依據 |
| :--- | :--- | :--- | :--- |
| **用途** | **控制硬體運作**。 | 讓用戶可進行**特定工作**。 | |
| **相依性** | 管理系統資源，並為應用軟體提供平台。 | 在系統軟體提供的平台上運作。 | |

### 3.2 系統軟體

#### A. 操作系統 (Operating System, OS)

OS 控制硬體運作，並與驅動程式和應用軟體互動。

**OS 主要功能**：

*   **處理程序管理**：安排 CPU 執行或取消處理程序。
*   **記憶體管理**：為處理程序分配 RAM 記憶體。
*   **周邊設備管理**：藉驅動程式控制周邊設備的輸入和輸出。
*   **檔案系統管理**：建立檔案系統以整理資料夾和檔案。
*   **硬碟管理**：控制儲存空間格式化和分區。
*   **用戶界面 (User Interface)**：於屏幕展示資訊，並讓用戶與電腦互動。

**用戶界面類型**：

| 類型 | 描述 | 來源依據 |
| :--- | :--- | :--- |
| **圖形用戶界面 (GUI)** | 用戶以視窗、圖示、選單和按鈕等圖形元素與系統互動；對新用戶而言較容易，常用於滑鼠、觸控式屏幕。 | |
| **命令行界面 (CLI)** | 用戶以一系列特定的**文字指令**操作系統；需要較少系統資源，運作較 GUI 快；**不需要定點設備**（滑鼠）。 | |

#### B. 驅動程式 (Driver)

驅動程式用作翻譯成機器層面的指令，以控制周邊設備，讓操作系統能夠控制特定的周邊設備。

#### C. 實用程式 (Utility Programs)

實用程式是維護電腦系統並改善其效率的軟體，例如：

*   **檔案管理程式**：處理檔案的展示、新增、複製或刪除。
*   **系統監察軟體**：監察 CPU、記憶體用量和網絡流量。
*   **數據壓縮程式**：將多個檔案壓縮成一個壓縮檔（如 zip, 7z），減少檔案大小，利於傳輸和儲存。
*   **磁碟重組軟體** (Defragmentation)：將分散儲存的檔案數據重新整理，使其連續儲存，以提高**硬碟**的數據存取速度（**固態硬碟 (SSD) 不需要磁碟重組**）。

### 3.3 應用軟體 (Application Software)

應用軟體主要分為：

1.  **生產力軟體 (Productivity Software)**：目的為提升工作生產力（如文書處理、試算表、簡報軟體、數據庫管理）。
2.  **通訊軟體 (Communication Software)**：用於網絡上用戶之間交換資訊（如瀏覽器、電子郵件用戶端、即時通訊程式）。
3.  **教育軟體 (Educational Software)**：用於獲取知識或輔助教育工作者（如百科全書、學習詞典、學科計算軟體）。

### 3.4 運作方式 (Modes of Operation)

不同的電腦系統可以採用不同的運作方式來處理數據。

| 運作方式 | 描述 | 應用舉例 | 來源依據 |
| :--- | :--- | :--- | :--- |
| **成批工作處理 (Batch Processing)** | 數據收集後**不會即時處理**，而是隔一段特定時間集中處理。 | 銀行製作月結單、定期備份、學生成績單印刷。 | |
| **實時處理 (Real-time Processing)** | 系統**接收數據後馬上處理**，並使用戶得到最貼近現況的即時響應。 | 售票系統、航空訂位、自動櫃員機 (ATM)（筆記未涵蓋，但屬於實時處理的常見例子）。 | |
| **並行式處理 (Parallel Processing)** | 一項工作會分為多個子工作，並交由**多個處理器**同時執行，以提升效率。 | 處理器通常共用記憶體。 | |
| **分散式處理 (Distributed Processing)** | 一個程式或一部電腦負責管理多部電腦的互動，透過**網絡**將工作分派給**不同電腦**處理。 | 處理視覺效果渲染、大型科學運算。 | |
| **虛擬化 (Virtualisation)** | 藉助實用程式將電腦硬體分成多個**虛擬機器**，可以運行與主機不同的操作系統，節省硬體成本。 | 運行不兼容的軟體，建立多個獨立的測試環境。 | |

**總結**

作為電腦系統的基礎，**硬體**（如 CPU, RAM, 輔助儲存器）提供了運算和儲存的能力，而 **系統軟體**（如 OS, 驅動程式）則充當了管理員和翻譯者，讓**應用軟體**得以順利運作，並透過 **輸入/輸出設備** 與用戶和現實世界互動。這種分層結構確保了電腦系統的穩定性、效率和多功能性。
