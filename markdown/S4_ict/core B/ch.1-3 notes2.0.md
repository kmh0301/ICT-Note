# 資訊及通訊科技（ICT）教科書B冊 完整課堂筆記

## 第 1 章：輸入及輸出設備 (Input and Output Devices)

### 1.1 輸入-處理-輸出周期 (IPO Cycle)

**學習目標**：描述輸入、處理與輸出之間的關係。

**核心概念**：
將數據轉換為資訊需要三個步驟，這個過程稱為 **輸入-處理-輸出周期/循環 (IPO Cycle)**。

1.  **輸入 (Input)**：將**數據**送入電腦。
2.  **處理 (Processing)**：程式處理所輸入的數據，儲存的數據亦可在這步驟被提取或更新。
3.  **輸出 (Output)**：電腦輸出有意義的**資訊**。

**互動與練習**

**討論題 (IPO 辨識)**：
以電子試算表計算學生平均分數為例，哪些是「數據」？哪些是「資訊」？
*   學生個別的原始分數：**數據** (輸入)。
*   全班平均分和圖表：**資訊** (輸出)。

### 1.2 輸入設備 (Input Devices)

輸入設備負責 IPO 循環的第一步，可向電腦輸入指令，或從現實世界中擷取數據。

#### A. 連接方式

輸入設備可以是有線或無線連接。
*   **有線連接**：使用通用串列匯流排 (USB) 連接埠。
*   **無線連接**：使用射頻 (RF) 技術（如 2.4 GHz 連接）、**藍牙 (Bluetooth)** 或 **Wi-Fi 連線**。

### 1.3 輸出設備 (Output Devices)

輸出設備負責 IPO 循環的最後一步，輸出實體或數碼格式的資訊。

#### A. 顯示器 (Monitors)

顯示器是電腦相當重要的輸出設備。

**顯示器類型比較**：

| 類型 | 運作方式 | 優點 | 來源依據 |
| :--- | :--- | :--- | :--- |
| **液晶體顯示器 (LCD)** | 使用冷陰極螢光燈 (CCFL) 或發光二極管 (LED) 作為背光。 | 價格較低；屏幕尺寸選擇多。 | |
| **有機發光二極管 (OLED)** | **OLED 為光源**，**不需要背光**。 | 對比度更高；亮度更一致；視角更廣；更輕薄。 | |

**顯示器規格**：

*   **解像度 (Resolution)**：屏幕上有多少點/像素。
*   **像素密度 (Pixel Density)** (ppi)：指指定範圍中有多少像素。
*   **更新率 (Refresh Rate)** (Hz)：屏幕在特定時間內更新影像的次數。
*   **連接埠**：VGA、DVI、**HDMI**（支援音訊，解像度高）和 **DisplayPort**（支援音訊，解像度高）。

**討論題 (顯示器比較)**：
為何 OLED 顯示器在解像度相似的情況下，通常影像品質仍優於傳統 LCD？
*   **OLED** 每個像素獨立發光，可以完全關閉，因此**不需要背光**，能實現**極致的對比度**。

#### C. 打印機 (Printers)

| 屬性 | 熱感式打印機 (Thermal) | 噴墨打印機 (Inkjet) | 激光打印機 (Laser) | 來源依據 |
| :--- | :--- | :--- | :--- | :--- |
| **購價/成本** | 較便宜 / 每頁成本較便宜。 | 較貴 / 每頁成本較昂貴。 | **非常昂貴** / 每頁成本較便宜。 | |
| **速度** | 較快。 | 較慢。 | 較快。 | |
| **耐用性/備註** | **不適合長期文件**（易褪色）。常見於收據。 | 打印成品遇水可能會變模糊。 | **不易受水影響**，成品耐用。 | |

**應用情景題 (打印機選擇)**
若需列印保養單據，預計存放數年，應選用哪種打印機？
*   應選用**激光打印機**。因為熱感式打印品的油墨不耐用，**容易褪色**，不適合長期保存。

### 1.4 電腦應用

**銷售點終端機 (POS Terminals)**：常見於電子支付和零售業。由多個設備組成，包括**磁帶卡閱讀器/智能卡閱讀器、數字鍵盤、條碼掃描器**，以及**收據打印機**（通常為熱感式）。

**應用範疇**：
*   **商業用途**：視像會議、收銀服務、電子支付系統、自助終端機。
*   **教育用途**：網上課程、圖書館借還、博物館展覽。

## 第 2 章：電腦硬件 (Computer Hardware)

### 2.1 主機 (Main System)

#### A. 主機板 (Motherboard)

主機板連接電腦各部件的**平台**。連接的設備包括：主機記憶體 (RAM/ROM)、中央處理器 (CPU)、輔助儲存設備 (HDD/SSD) 等。

#### B. 處理器 (Processor)

1.  **中央處理器 (CPU)**：電腦的「大腦」，控制電腦系統的運作。
2.  **圖形處理器 (GPU)**：專門負責圖像處理，有更多核心，多工作業能力較強。
    *   **獨立圖形卡 (Graphics Card)**：圖像處理能力較高，適用於電腦輔助設計、剪輯影片或電腦遊戲。
    *   **集成圖形處理器 (iGPU)**：集成於 CPU 內，**成本較低**，消耗較少能源，適用於一般文書處理。

**討論題 (GPU 應用)**
為什麼訓練人工智能時，通常使用 **GPU** 作為核心？
*   GPU 擅長**同時進行大量簡單運算**；CPU 擅長處理複雜的任務。由於 AI 深度學習需要快速分析大量數據，因此 GPU 成為核心。

#### E. 匯流排 (Bus System)

電腦系統使用匯流排/總線傳輸數據和控制訊號，連接各部件。

| 匯流排類型 | 傳輸的物件 | 方向 | 來源依據 |
| :--- | :--- | :--- | :--- |
| **控制匯流排** | 控制訊號 | **單向** (處理器發送，不接收)。 | |
| **位址匯流排** | 記憶體位址 | **單向** (由處理器傳輸至不同部件)。 | |
| **數據匯流排** | 數據或指令 | **雙向** (發送和接收)。 | |

**練習題 2.1 (匯流排傳輸)**
**（源自測試站 2.1 Q3）**
控制匯流排將**控制訊號**由處理器傳輸至不同部件。

### 2.2 中央處理器 (CPU)

#### E. 機器周期 (Machine Cycle)

CPU 執行指令的過程包含四個主要階段：
1.  **提取 (Fetch)**：從記憶體中獲取指令。
2.  **譯碼 (Decode)**：將指令譯碼，由 **控制部件 (CU)** 負責。
3.  **執行 (Execute)**：執行指令，由 **算術及邏輯運算部件 (ALU)** 負責。
4.  **儲存 (Store)**：將處理結果儲存在**累加器**等寄存器中。

**練習題 2.2 (機器周期)**
**（源自測試站 2.2 Q1）**
在機器周期中，**控制部件 (CU)** 負責譯碼。

#### F. CPU 規格

*   **時鐘頻率 (Clock Rate)**：CPU 在一秒內進行的時鐘周期數目，單位為赫茲 (Hz) 或 千兆赫 (GHz)。
*   **核心數 (Core)**：指 CPU 內的處理器單元數目，決定可同時執行的程序數目。
*   **字長 (Word Length)**：CPU 每次處理的位元數目，通常為 32 位元或 **64 位元**。

---

### 2.3 儲存設備 (Storage Devices)

#### A. 主記億體 (Main Memory)

| 類型 | 易失性 (Volatility) | 可重複性 (Re-programmability) | 用途 | 來源依據 |
| :--- | :--- | :--- | :--- | :--- |
| **RAM** (隨機存取記憶體) | **易失性**（關機數據消失）。 | 可以重複讀寫。 | 暫存數據供 CPU 處理。 | |
| **ROM** (唯讀記憶體) | **非易失性**（數據保存）。 | 通常不可重複寫。 | 儲存**啟動電腦所需指令** (BIOS)。 | |

*   **快取記憶體 (Cache Memory)**：位於 CPU 中，存取速度比 RAM 快，用於暫存最常存取的數據。

**練習題 2.3 (RAM 特性)**
**（源自測試站 2.3 Q1）**
RAM 的數據存取速度高於其他輔助儲存設備 (1)；其數據於電腦關機時消失 (3)。
*   *備註：RAM 的存取速度**低於** CPU 快取 (2)。*

#### B. 輔助儲存器 (Secondary Storage)

所有輔助儲存器都是**非易失性**的。

**存取方法**：
*   **順序存取 (Sequential Access)**：必須先處理所有項目才能存取特定數據（例如：**磁帶**）。
*   **直接存取 (Direct Access)**：可直接存取任何儲存位（例如：**硬碟 (HDD), 固態硬碟 (SSD), 光碟**）。

**輔助儲存器比較 (表 2.14)**：

| 屬性 | 磁帶 | 硬碟 (HDD) | 固態硬碟 (SSD) | 光碟 (Optical) | 來源依據 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **存儲容量** | **最大**。 | 大。 | 中。 | 小。 | |
| **存取方法** | 順序存取。 | 直接存取。 | 直接存取。 | 直接存取。 | |
| **數據存取速度** | **最低**。 | 中。 | **高**。 | 中。 | |
| **耐用度** | 易受潮損。 | 易受撞擊損壞。 | **耐用**。 | 刮花時容易受損。 | |

#### C. 記憶體階層 (Memory Hierarchy)

儲存設備的位置越高，其數據存取速度越高，儲存容量越小，每容量價格越高。

**速度排名 (快至慢)**：
**寄存器** → **處理器的快取記憶體** → **RAM (主記憶體)** → **SSD** (輔助儲存器) → **硬碟** (輔助儲存器) → **磁帶** (輔助儲存器)。

**練習題 2.4 (記憶體排名)**
**（源自測試站 2.4 Q1）**
將以下記憶體體積按照數據存取速度由**快至慢**排列：(1) 快閃記憶體、(2) 快取記憶體、(3) 寄存器。
**答案：(3) > (2) > (1)**。

## 第 3 章：電腦軟體 (Computer Software)

### 3.1 電腦系統

軟體是數據或指令，用作協助電腦完成特定工作。

| 屬性 | 系統軟體 (System Software) | 應用軟體 (Application Software) | 來源依據 |
| :--- | :--- | :--- | :--- |
| **用途** | **控制硬體運作**。 | 讓用戶可進行**特定工作**。 | |
| **例子** | Microsoft Windows, macOS。 | Microsoft Excel, Adobe Photoshop。 | |

### 3.2 系統軟體

#### A. 操作系統 (Operating System, OS)

OS 控制硬體運作，並與驅動程式和應用軟體互動。

**OS 主要功能**：
包括：**處理程序管理** (安排 CPU 執行程序)、**記憶體管理** (分配 RAM)、**周邊設備管理**、**檔案系統管理**、**硬碟管理**、**用戶界面**和**多工處理**。

**用戶界面類型**：

| 類型 | 特點 | 來源依據 |
| :--- | :--- | :--- |
| **圖形用戶界面 (GUI)** | 用戶以視窗、圖示、選單、按鈕等圖形元素互動。對新用戶較容易，通常需用**定點設備**（如滑鼠）。 | |
| **命令行界面 (CLI)** | 用戶以**文字指令**操作系統。運作較 GUI 快，**不需要定點設備**。 | |



---

#### B. 實用程式 (Utility Programs)

實用程式用於維護電腦系統並改善其效率。

| 實用程式 | 功能 | 來源依據 |
| :--- | :--- | :--- |
| **檔案管理程式** | 處理檔案的展示、新增、複製或刪除。 | |
| **數據壓縮程式** | 將多個檔案壓縮，如 zip, 7z。**好處**是減少檔案大小，利於傳輸和儲存。 | |
| **磁碟重組軟體** | 將分散儲存的檔案數據重新整理，使其連續儲存。目的：提高**硬碟**的數據存取速度。 | |
| **備份軟體** | 建立檔案和系統設定的備份。 | |

**討論題 (SSD 與重組)**
固態硬碟 (SSD) 通常被認為**不需要**進行磁碟重組，為什麼？
*   SSD 使用快閃記憶體，其存取是**直接存取**，讀取分散的數據塊對存取速度的影響極小。磁碟重組軟體不適用於 SSD。

#### C. 驅動程式 (Driver)

驅動程式用作翻譯成機器層面的指令，以控制周邊設備，讓操作系統能夠控制特定的周邊設備。

---

### 3.4 運作方式 (Modes of Operation)

| 運作方式 | 描述 | 應用舉例  |
| :--- | :--- | :--- |
| **成批工作處理 (Batch Processing)** | 數據收集後**不會即時處理**，而是隔一段特定時間集中處理。 | 銀行製作月結單、定期備份、學生成績單印刷。 | 
| **實時處理 (Real-time Processing)** | 系統**接收數據後馬上處理**，並使用戶得到最貼近現況的即時響應。 | 售票系統、航空訂位。 | 
| **並行式處理 (Parallel Processing)** | 一項工作由**多個處理器**同時執行，共享記憶體。 | 
| **分散式處理 (Distributed Processing)** | 透過**網絡**將工作分派給**多部電腦**處理，記憶體不共享。 | 處理視覺效果渲染。 | 
| **虛擬化 (Virtualisation)** | 藉助實用程式將電腦硬體分成多個**虛擬機器**，可以運行與主機不同的操作系統，節省硬體成本。 | 

**應用分析題 (處理模式)**
**（源自測試站 3.2 Q1）**
為何電影院的**網上購票系統** 必須使用**實時處理**？
*   **實時處理**：顧客選擇座位時，系統會**馬上處理**該訂購指令，並立即更新座位供應狀況。這可以確保在同一時間內，多個顧客不會購買到已被佔用的座位，避免超額預訂。

## 總結

如果將電腦系統比喻為一所**行政辦公室**：
*   **硬體 (Hardware)** 就像辦公室的實體設施：**CPU** 是經理 (處理指令)；**RAM** 是經理的臨時工作台 (暫存數據)；**輔助儲存器** 是檔案櫃 (長期儲存數據)。
*   **系統軟體 (System Software)** 就像辦公室的行政程序和規範 (OS) 或翻譯員 (Driver)，確保經理和員工（應用軟體）能使用設施 (硬體)。
*   **應用軟體 (Application Software)** 就像員工們實際處理的特定任務 (如文書工作、會計、通訊)。
*   **輸入設備 (Input)** 就像接收到的信件和電話 (數據輸入)；**輸出設備 (Output)** 就像寄出的報告或列印的檔案 (資訊輸出)。
