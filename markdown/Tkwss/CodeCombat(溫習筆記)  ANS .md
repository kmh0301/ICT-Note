# CodeCombat Python 編程筆記 (玩法階段重點)

在 CodeCombat 中，您的目標是使用 **Python** 語言來控制英雄（`hero`）完成任務。以下是按遊戲進度整理的編程重點與實用技巧：

## I. 關卡前期（移動與方向控制）1－6關

這是遊戲最基礎的階段，您主要學習控制英雄物件（`hero`）的基本動作。

### 1. 基本移動指令 (Movement Commands)

你需要調用英雄（`hero`）所擁有的**方法（指令）**來讓它移動。

*   **語法結構：** `物件.方法(參數)` 
*   **示例：** `hero.moveRight(3)` 

### 2. 使用參數控制距離 (Parameters)

*   **移動指令：** 許多移動指令需要在括號 `()` 內提供額外的資訊——**參數**，用來告訴程式如何執行該動作 。
*   **關鍵提示：** 遊戲中的關卡注釋可能會提醒你「**使用帶有參數的移動語句來移動更遠並收集寶石**」。例如，`hero.moveRight(3)` 表示向右移動 3 步/距離 。

### II. 關卡中期（互動、攻擊與文字）7－12關

當你遇到敵人或需要解鎖密碼門時，就需要學習處理**字串 (Strings)**。

#### 3. 說話或攻擊：文字必須加引號！

當您需要讓英雄**說話**或指定**攻擊目標**的名字時，這些文字資訊必須被視為「字串」（String）。在 Python 中，字串必須使用**引號**（單引號 `''` 或雙引號 `""`）包圍，否則出現語法錯誤無法執行代碼。

| 動作 | 範例 | 必須注意的細節 |
| :--- | :--- | :--- |
| **說出密碼** | `hero.say("Achoo")`  | 如果缺少引號，Python 會認為 `Achoo` 是一個變數，導致程式出錯。 |
| **攻擊敵人** | `hero.attack("Two")`  | 敵人名字如 `Two` 必須用引號包住 。 |

---

### III. 循環結構：讓英雄重複行動 (Loops for Repetition)

在 CodeCombat 中期關卡中，當你需要英雄**重複執行**一連串動作（例如不斷地繞圈移動或收集寶石）時，就需要用到**循環結構**。

*   **目前的重點：無限循環 (`while True`)**
    由於您目前接觸的關卡通常需要英雄持續動作直到任務完成，我們使用「無限循環」：
    ```python
    while True:
        # 在這裡放入所有需要重複的動作
    ###我不是循環的一部分
    ```
    `while True:` 的意思就是：**只要條件是「真」（True），就一直重複執行下面的程式碼。** 


*   **執行關鍵：縮排**
    只有在 `while True:` **下方並向右縮排**的程式碼，才會被電腦視為「屬於這個循環，需要重複執行的步驟」。

| 動作 | 執行效果 |
| :--- | :--- |
| **正確縮排** | 這些指令將會不斷重複執行。 |
| **錯誤（無縮排）** | 指令將只執行**一次**，然後程式就可能結束或出錯。 |

*   **實用快捷鍵 (用於縮排)：**
    *   你可以使用鍵盤的 **TAB 鍵**來快速進行一次縮排 。
    *   或者手動按下 **4 下空白鍵**來完成縮排 。



## IV. 實用技巧與通關提醒

### 編程效率提示

1.  **程式碼提示**：在輸入程式碼時，你可以嘗試輸入相關函數的前幾個字母，讓系統彈出相關函數供你選擇，這樣就不用完整打出函數名稱（此為一般編程環境的常見功能，未在來源中提及具體 CodeCombat 的細節，但仍是實用技巧）。
2.  **快捷鍵**：
    *   使用 **CTRL + C & CTRL + V** 鍵組合來執行複製貼上的功能，將剪貼簿中的內容貼上 。
    *   使用 **CTRL + Z** 鍵組合來還原（Undo）上一步的動作。

### 關卡理解與除錯輔助
1.  **物件的重要性**：所有動作都必須由**誰**（即 `hero` 物件）來執行。因此，指令必須以 `hero.` 開頭 。
2.  **勿忘括號 `()`**：即使函數不需要參數，也必須加上括號 `()` 來執行該動作，例如 `hero.moveUp()` 。
3.  **大小寫敏感**：Python 區分大小寫，所以 `hero.moveRight(3)` 和 `Hero.moveright(3)` 會被視為不同的東西。英雄物件應使用小寫的 `hero`。
