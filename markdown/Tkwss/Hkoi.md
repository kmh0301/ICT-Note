# HKOI 初級組 速成溫習筆記：數據結構與算法核心

## I. 競賽與格式概覽 (HKOI Overview)

*   **目標：** 訓練邏輯思維，解決涉及數學和數據處理的程式問題。
*   **試卷格式（初級組）：** 主要分為三部分：
    1.  **甲部 (一) True or False（真假題）：** 5 題。
    2.  **甲部 (二) Multiple Choice（多項選擇題）：** 20 題。
    3.  **乙部 Fill-in-the-blanks（填充題）：** 7 題（共 A - K 等空格）。
*   **作答核心：** 許多題目要求**追蹤程式碼 (Tracing Code)** 的執行流程、計算循環次數、或判斷邏輯運算結果。

## II. 核心概念 1：陣列與索引 (Array and Indexing)

**概念簡介：** 陣列 (Array/List) 是一種可以儲存大量同類型數據的結構。

| 概念 | 說明 |
| :--- | :--- |
| **陣列 (Array)** | 想像成一排**置物櫃**，用來儲存一串值（如學生成績）。在 Python 中通常稱為串列 (List)。|
| **索引 (Index)** | 每個位置的**編號**，用於存取特定的值。在 C/C++ 和 Python 中，索引通常**從 0 開始**。|
| **錯誤陷阱** | 必須注意**索引越界 (Index Out of Bounds)** 的錯誤，例如長度為 N 的陣列，最大的有效索引是 $N-1$。|

**💡 HKOI 試題引導：程式碼追蹤**

*   **範例 (參考 HKOI 2021/22 Q17)：**
    陣列 `a` 定義為 `a = {2, 0, 2, 1, 1, 1, 1, 3}`。
    要找出 `a` 的值，必須從索引 0 開始數：
    | 索引 | 0 | 1 | 2 | 3 | **4** | 5 | 6 | 7 |
    | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
    | 值 | 2 | 0 | 2 | 1 | **1** | 1 | 1 | 3 |
    因此，`a` 的值是 1。精確追蹤索引是解題的基礎。

### III. 核心算法 2：檢索 (Searching)

檢索（搜尋）是指在數據集中尋找特定值的過程。

#### A. 線性檢索 (Linear Search)

*   **概念：** 從陣列的**開頭到結尾，逐一檢查**每個元素。
*   **前提：** **不需要**將陣列排序。
*   **應用：** 常用於尋找最大值或最小值。
*   **效率：** 當數據量 $N$ 很大時，效率較低，因為在最壞情況下需要檢查 $N$ 個元素。

*   **💡 HKOI 試題引導 (參考 HKOI 2023/24 Q6):**
    程式碼嘗試找出陣列中的最大值 (`maxa`) 和最小值 (`mina`)。線性檢索的邏輯就是不斷更新最大值/最小值。
    ```c++
    // 陣列 a = {3, 1, 4, 1, 5}; 
    int mina = 0, maxa = 0; // 陷阱：初始值設定不當
    for (int i = 0; i < 5; ++i) { 
        if (a[i] > maxa) { maxa = a[i]; } // 找到更大的，更新
        if (a[i] < mina) { mina = a[i]; } // 找到更小的，更新
    } // 由於 a[i] 均 >= 0，且 mina 初始為 0，所以 mina 始終是 0。
    ```

---
#### B. 二分檢索 (Binary Search)

*   **概念：** 每次比較中間點 (Mid)，將搜尋範圍**砍半**，極大地提高效率。
*   **前提：** **陣列必須是已排序 (Sorted) 的**。
*   **效率：** 在大型數據集上表現卓越，因為它以對數時間 (Logarithmic Time) 運行。
*   **應用範圍：** 適用於按字典序排列的字串陣列，以及包含重複元素的排序陣列。

*   **💡 HKOI 試題引導 (HKOI 2021/22 Q4):**
    陳述：「在判斷一個元素是否存在於至少有 10 個元素及經排序的陣列中時，二分檢索法所需要的比較次數**總是**比線性檢索法所需要的少。」
    *   **答案：** 假 (False)。
    *   **原因：** 如果目標元素恰好在陣列的第一個位置，線性檢索只需 1 次比較。因此，二分檢索**不總是**最快的，但在絕大多數情況下，尤其是數據量大時，它遠比線性檢索高效。

### IV. 核心算法 3：排序 (Sorting)

排序是將數據按特定順序（通常為遞增或遞減）排列的過程。

| 類型 | 概念 | HKOI 相關考點 |
| :--- | :--- | :--- |
| **比較排序** (Comparison Sort) | 透過比較元素大小來進行排序。包括冒泡、選擇、插入。| 常用於程式碼追蹤，要求理解交換和迴圈邏輯。 |
| **冒泡排序** (Bubble Sort) | 透過**重複相鄰元素的比較與交換**，將大（或小）的元素逐步「冒泡」到正確位置。 | 在 HKOI 2024/25 P1 Q1(a) 中，要求完成其雙重迴圈的結構。它**可能**會重複比較同一對元素兩次。|
| **插入排序** (Insertion Sort) | 每次將一個未排序元素插入到已排序部分的正確位置。| **不會**重覆比較同一對元素兩次。|
| **快速排序** (Quicksort) | 效率高，通過選擇**樞紐 (Pivot)** 將陣列分割並遞歸排序。| **不會**重覆比較同一對元素兩次。|
| **相鄰交換** (Adjacent Swaps) | 將序列升序排列所需的**最少**相鄰交換次數，等於序列中的**逆序對 (Inversions)** 數量。HKOI 2021/22 Q15 考查了此點。|

### V. 必備技能：循環、條件與除錯 (Looping, Conditionals & Debugging)

競賽中大量使用控制結構 (Control Structures)，如條件語句 (`if-else`) 和循環語句 (`for`, `while`)。

#### A. 程式碼追蹤 (Tracing Code)

*   **巢狀迴圈 (Nested Loops):** 追蹤多層迴圈（如 $i, j, k$ 循環）的計數變量 (`cnt` 或 `ans`) 是常見題型。
*   **範例 (參考 HKOI 2022/23 Q13):**
    ```c++
    // for (int i = 0; i < 10; ++i) { 
    //   for (int j = 0; j < 10; ++j) { 
    //     if (i == j) break; // 陷阱：break只跳出內層迴圈
    //     ++cnt;
    //   } 
    // } 
    ```
    當 $i=j$ 時，內層循環立即停止，但外層循環繼續。最終的計數結果是 $0 + 1 + 2 + \dots + 9 = 45$。

#### B. 錯誤與除錯 (Error and Debugging)

*   **邏輯錯誤 (Logic Errors):** 程式可以成功編譯，但執行結果不正確。這是競賽中最常考的錯誤類型之一。
*   **索引越界 (Index Out of Bounds):** 當程式嘗試存取陣列範圍外的元素時發生。
*   **刪除元素陷阱 (Removing Elements):** 在循環中從陣列中刪除元素時，如果沒有正確處理索引 `i`，可能會跳過檢查下一個元素。
    *   **解決方法 (HKOI 2021/22 Section B Q6):** 在刪除（或移動元素）之後，必須確保當前索引 `i` 不會前進，以便檢查剛剛移動到 `a[i]` 位置的新元素。修正方法是在刪除元素後（`n--` 或 `dec(n)` 之後）增加一行程式碼，使 `i` 保持不動（例如 `i--` 或 `dec(i)`）。
*   **數據類型溢出 (Overflow):** 數值運算結果超出數據類型（如 32-bit `int`）的極限，會導致意外結果（例如，從正最大值繞回到負最小值）。

### VI. 進階概念提示 (Advanced Concepts Hint)

這些概念在兩堂速成班中無法詳細講解，但應當**知道它們的存在**，並留待課後自學。

1.  **進階排序：** 快速排序 (Quicksort)、合併排序 (Merge Sort)、計數排序 (Counting Sort)。它們針對不同情境提供更高的效率。
2.  **抽象數據結構：** 堆疊 (Stack) 和隊列 (Queue)。
    *   **堆疊 (Stack):** 後進先出 (LIFO)。在 HKOI 2024/25 Q6 中，最後推入的 5 是頂部 (Top)。
    *   **隊列 (Queue):** 先進先出 (FIFO)。在 HKOI 2024/25 Q6 中，最先推入的 3 是前端 (Front)。
3.  **位元運算 (Bitwise Operations):** HKOI 頻繁測試位元邏輯（如 `XOR`、`AND`、`OR`）。例如 `a XOR b = 0` 邏輯上等價於 `a = b`。
